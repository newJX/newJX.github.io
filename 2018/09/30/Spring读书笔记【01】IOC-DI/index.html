Spring读书笔记【01】IOC-DI

发表于 2018-09-30 | 分类于 Spring读书笔记

关于 IOC 和 DI

IOC 是指将 bean 的创建工作交由 Spring 来完成，而不是传统的在哪用就在哪手动 new

DI 是在 IOC 的前提下，用于配置 bean 之间的依赖关系

Spring IOC & DI 的三种方式

一、隐式的 bean 发现和自动装配

1) 组件扫描[IOC]

通过@component声明类来通知 Spring 创建指定的 bean，Spring 会扫描带@component的类(组件)，但组件扫描默认不开启，可以通过@Configuration声明一个配置类，@ComponentScan开启组件扫描。默认会扫描配置类所在包及其子包

也可以通过 xml 方式开启组件扫描(applicationContext.xml)

2) 自动装配[DI]

在@component声明的 bean 中，可以通过在构造器、setter 方法或其它普通方法上使用@Autowired来实现自动注入依赖

二、通过 java 显式配置

1) IOC

通过Configutation声明配置类，但不使用@ComponentScan开启组件扫描，而是在配置类中通过@Bean声明方法，该方法返回的对象会自动注册成为 Spring 上下文中的 bean

2) DI

在@Bean声明的方法中，如果要生成的 bean 依赖其他 beanA，可以在方法的入参增加 beanA，然后在方法体内使用构造器或 setter 方式注入，此种方式不要求 beanA 必须在 JavaConfig 中显式配置，但必须有通过其他方式实现了 IOC。也可以直接调用 JavaConfig 中@Bean声明的生成 beanA 的方法来实现，此种方式要求 beanA 必须在 JavaConfig 中显式配置

三、通过 xml 显式配置

1) IOC

创建 xml 文件，通过声明一个 bean，此处不需要开启组件扫描，也不需要在 bean 上加@component

2) DI

主要分为构造器注入和 setter 注入，可注入 bean、字面量、集合等
